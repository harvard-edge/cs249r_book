The Periodic Table of Machine Learning Systems

Two fundamental axes — abstraction layer and information-processing role — organize ML concepts the way electron shells and valence organize chemistry.

The Two Axes

↓ Rows: Abstraction Layer

Like electron shells, each layer builds on and contains the ones above. You can't have optimization without a model, or deployment without hardware.

1. Math — primitives
2. Algorithms — building blocks
3. Architecture — structural patterns
4. Optimization — efficiency techniques
5. Runtime — systems infrastructure
6. Hardware — physical compute
7. Production — deployment & ops

→ Columns: Information-Processing Role

From computer architecture and systems theory — five roles that exist in any information-processing system.

Represent — encodes, stores, structures
Compute — transforms information
Communicate — moves between components
Control — decides what happens
Measure — observes without transforming

The same-column test: Concepts in the same column genuinely behave alike.

Represent: Tensor, Probability, Parameter, Embedding, Topology, Hidden State, Optimizer State, Caching, Checkpointing, SRAM, DRAM, Blob Storage — all hold and structure information across all 7 layers.

Compute: Operator, Activation, Dense Dot, Convolution, Pooling, Attention, Routing, Quantization, Fusion, Tiling, Compilation — all transform inputs to outputs.

Communicate: Chain Rule, Backprop, Tokenization, Skip/Res, Distillation, Weight Averaging, Pipelining, Sync, Prefetching, Interconnect, RPC Protocol — all move information between components without deciding what to do with it.

Control: Objective, Constraint, Grad Descent, Search, Initialization, Masking, Scheduling, Regularization, Allocation, Arbiter, Load Balancer, Telemetry — all make decisions that govern system behavior.

Measure: Entropy, Loss Function, Receptive Field, Info Density, Throughput, Energy, Latency — all observe without changing. Noble gases.
